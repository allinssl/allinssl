# 域名详情 - 实名认证标签页需求说明（基于 NaiveUI ）

## 一、页面整体架构

页面为域名详情页的子模块，依托**域名详情页基础框架**，聚焦“实名认证”标签内容展示。整体采用「顶部导航 + 标签切换 + 区块化内容」布局，基于 NaiveUI 组件构建，实现实名认证状态、提示信息及认证数据的清晰呈现。

## 二、返回导航区

### 组件与交互

使用 `NButton` 组件，配置 `text` 类型（简化按钮样式，仅保留文字与图标），搭配 `NIcon` 组件（选用向左箭头图标，如 `chevron-left` ）。按钮文案为**“返回域名列表”** ，点击时触发路由跳转，回到域名列表页面，实现页面级导航。

## 三、页面标题区

### 标题展示

- **主标题**：显示当前域名，如 `example.com` ，使用 `NText` 组件，设置较大字号、加粗样式，突出域名主体。
- **副标题**：呈现“域名详细信息和管理” ，通过 `NText` 组件以常规字号、浅色字体展示，辅助说明页面功能定位。

## 四、标签页导航区

### 组件与交互

采用 `NTabs` 组件承载标签切换逻辑，包含**基本信息、实名认证、域名解析、操作日志** 四个 `NTabPane` 子组件。当前激活标签为“实名认证”，通过 `NTabs` 的 `activeKey` 属性控制选中状态，点击其他标签可切换至对应内容区，实现多维度域名信息的分类查看。

## 五、实名认证标签内容区

### 整体容器

使用 `NCard` 组件包裹所有内容，配置 `bordered` 属性（显示边框），为内容提供规整的视觉容器。借助 `NGrid` 组件（或 `NRow` + `NCol` 组合）进行布局，按“状态提示 → 警告说明 → 认证信息”的逻辑，纵向划分内容区块，保证页面结构清晰。

### 模块 1：实名认证状态

- **组件组合**：`NIcon`（选用勾选圆形图标，设置 `type="success"` ，呈现绿色样式 ）与 `NText` 组件组合。
- **内容展示**：
  - 标题：“实名认证已通过” ，`NText` 组件设置加粗、较大字号，突出认证结果。
  - 时间：“认证时间：2023-12-16 10:30:25” ，`NText` 组件以常规样式展示，补充认证时间维度信息。
- **布局方式**：通过 `NRow` 组件横向排列图标与文字，让状态结果直观呈现。

### 模块 2：警告提示

- **组件选择**：`NAlert` 组件，配置 `type="warning"` ，显示橙色警告样式，同时开启 `showIcon` 属性（展示警告图标 ）。
- **文案内容**：
  - 第一行：“请确保实名认证信息完整且准确，否则可能影响域名归属及使用”
  - 第二行：“实名认证完成后至少 72 小时才可进行网站备案，详情请咨询相关备案平台。”
- **交互逻辑**：可配置 `closable` 属性，控制是否显示关闭按钮，默认展示提示信息，无复杂交互，纯信息告知。

### 模块 3：认证信息卡片

- **容器组件**：嵌套 `NCard` 组件，设置 `title` 为“认证信息” ，作为信息区块的标题。内部使用 `NGrid` 组件，配置 `cols="2"` ，将内容划分为两列布局，实现字段的左右对齐展示。
- **内容结构**：
  | 左侧字段 | 右侧字段 | 展示组件 | 说明 |
  |------------|------------|----------|--------------------------|
  | 认证类型 | 证件类型 | `NText` | 左右两列对称布局 |
  | 个人认证 | 身份证 | `NText` | - |
  | 姓名 | 证件号码 | `NText` | - |
  | 张三 | 110****\*****1234 | `NText` | 证件号脱敏处理，隐藏中间部分 |
  | 联系电话 | 电子邮箱 | `NText` | - |
  | 138\***_5678 | zhang_**@example.com | `NText` | 电话、邮箱脱敏，保护隐私 |

## 六、数据渲染逻辑

- **状态与时间**：从后端接口获取 `authStatus`（认证状态，如“已通过” ）、`authTime`（认证时间戳，格式化后展示为 `2023-12-16 10:30:25` ），动态填充到对应 `NText` 组件。
- **警告文案**：可配置为固定提示内容（如产品侧要求的合规提示 ），直接通过 `NAlert` 的 `content` 属性设置；也支持从接口获取动态提示，根据业务需求灵活调整。
- **认证信息**：接口返回认证类型、证件类型、姓名、证件号等字段，通过 `NText` 组件渲染展示，对手机号、证件号、邮箱等敏感信息，按产品规则进行脱敏处理（如隐藏中间部分字符 ）后显示。

## 七、组件样式与细节优化

- **图标适配**：`NIcon` 组件的图标大小、颜色与整体页面风格适配，确保视觉统一。例如，认证状态的勾选图标，尺寸与文字协调，颜色与“已通过”状态匹配。
- **间距与排版**：各组件（如 `NCard` 、`NGrid` 内的 `NCol` ）通过 `style` 或 NaiveUI 内置的间距配置（如 `gutter` 属性 ），设置合理的内边距、外边距，保证内容不拥挤，提升可读性。
- **响应式适配**：借助 NaiveUI 组件的响应式能力，在不同设备屏幕（如移动端、桌面端 ）下，`NGrid` 布局自动调整列数或排版，确保内容展示完整、美观。

通过上述设计，可精准还原“实名认证”标签页的布局、交互与数据展示逻辑，基于 NaiveUI 组件体系，开发可高效实现页面搭建，保障功能与视觉效果的落地。
