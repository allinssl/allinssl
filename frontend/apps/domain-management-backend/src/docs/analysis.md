# 域名详情 - 域名解析标签页需求说明（基于 NaiveUI ）

## 一、页面整体架构

作为域名详情页的子模块，依托**域名详情页基础框架**，聚焦“域名解析”标签内容展示。采用「顶部导航 + 标签切换 + 解析记录管理」布局，基于 NaiveUI 组件构建，实现解析记录的添加、编辑、删除及状态展示。

## 二、返回导航区

复用域名详情页的返回导航逻辑：  
使用 `NButton`（`text` 类型 ） + `NIcon`（向左箭头 `chevron-left` ），文案为**“返回域名列表”** 。点击触发路由跳转，回到域名列表页，完成页面级导航。

## 三、页面标题区

与域名详情页保持一致：

- **主标题**：显示域名（如 `example.com` ），通过 `NText` 组件设置**大字号、加粗**，突出域名主体。
- **副标题**：“域名详细信息和管理” ，以 `NText` 组件**常规字号、浅色**展示，辅助说明页面功能。

## 四、标签页导航区

基于 `NTabs` 组件实现标签切换：

- 当前激活标签为**“域名解析”** ，通过 `NTabs` 的 `activeKey` 控制选中状态。
- 其他标签（基本信息、实名认证、操作日志 ）保留可切换能力，点击可跳转至对应内容区，支持多维度域名信息的分类查看。

## 五、域名解析标签内容区

### 整体容器

使用 `NCard` 组件包裹所有内容，配置 `bordered` 属性（显示边框 ），为解析记录管理提供规整的视觉容器。通过 `NGrid`（或 `NRow` + `NCol` ）实现布局，按“操作按钮 → 解析记录表单 → 记录列表” 纵向排列内容。

### 模块 1：添加解析按钮

- **组件**：`NButton`（`type="primary"` 蓝色按钮 ），文案为**“+ 添加解析”** 。
- **交互**：点击时动态渲染一条空白解析记录编辑行（或弹出添加弹窗 ），用于创建新的 DNS 解析记录。

### 模块 2：DNS 解析记录编辑/添加行

#### （1）输入项与交互

通过 `NRow` + `NCol` 实现**行内表单布局**，包含以下可交互组件：

- **主机记录**：`NInput` 组件，支持输入主机记录值（如 `@`、`www` ），`placeholder` 为**“主机记录”** 。
- **记录类型**：`NSelect` 组件，选项包含 `A`、`CNAME`、`MX`、`TXT` 等（与 DNS 解析类型对齐 ），`placeholder` 为**“记录类型”** 。
- **记录值**：`NInput` 组件，根据记录类型动态校验输入（如 `A` 类型需填 IP 地址 ），`placeholder` 为**“记录值”** 。
- **TTL**：`NSelect` 组件，选项包含 `10分钟`、`600`（10 分钟 ）、`3600`（1 小时 ）等常用值，`placeholder` 为**“TTL”** 。
- **状态标记**：`NTag` 组件，默认显示**“待保存”**（黄色标签 `type="warning"` ），标记记录未保存状态。
- **操作按钮**：
  - **保存**：`NButton`（`type="success"` 绿色 ），点击时调用接口提交解析记录，成功后更新状态为**“正常”** 。
  - **取消**：`NButton`（`type="error"` 红色 ），点击时移除当前编辑行（或清空输入内容 ），取消添加/编辑操作。

#### （2）辅助说明文案

在编辑行下方，通过 `NText` 组件展示**解析类型说明**（如 “A 记录 - 将域名指向 IPv4 地址” ），并补充示例（如 “主机记录填写 @ 或 www，记录值填写 192.168.1.1” ），降低用户操作门槛。

### 模块 3：DNS 解析记录列表

#### （1）列表表头

通过 `NTable` 组件实现表格布局，表头字段包含：

- 主机记录、记录类型、记录值、TTL、状态、操作  
  表头文字通过 `NText` 组件展示，清晰定义每列内容。

#### （2）列表数据行

每行数据从后端接口获取，动态渲染以下内容：

- **主机记录**：`NText` 展示记录值（如 `@`、`www` ）。
- **记录类型**：`NText` 展示类型（如 `A`、`CNAME` ）。
- **记录值**：`NText` 展示解析值（如 `192.168.1.1`、`mail.example.com` ）。
- **TTL**：`NText` 展示时长（如 `600`、`3600` ）。
- **状态**：`NTag` 组件，根据记录状态展示标签：
  - 正常：`type="success"`（绿色标签，文案**“正常”** ）。
  - 异常：`type="error"`（红色标签，文案**“异常”** ，需结合接口状态扩展 ）。
- **操作**：
  - **编辑**：`NButton`（`type="info"` 蓝色 ），点击时将当前行数据回显到“编辑行”，进入编辑状态。
  - **删除**：`NButton`（`type="error"` 红色 ），点击时调用接口删除记录，成功后从列表移除。

## 六、数据渲染与交互逻辑

### （1）初始化加载

进入“域名解析”标签页时，调用后端接口获取**已有解析记录列表**，通过 `NTable` 动态渲染数据行，展示当前域名的 DNS 解析配置。

### （2）添加解析记录

点击“添加解析”按钮，动态生成空白编辑行（或弹出 `NModal` 弹窗 ），用户填写主机记录、记录类型、记录值、TTL 后：

- 点击“保存”：校验输入合法性（如 IP 格式、域名格式 ），调用**添加解析记录接口**。成功后，更新列表数据，新增记录状态为“正常”；失败则提示错误信息（如 `NMessage` 组件弹出报错 ）。
- 点击“取消”：移除编辑行（或清空弹窗表单 ），放弃添加。

### （3）编辑解析记录

点击列表行“编辑”按钮，将记录数据回显到编辑行（或弹窗表单 ），用户修改后：

- 点击“保存”：调用**更新解析记录接口**，同步修改内容到后端，成功后更新列表状态；失败则提示错误。
- 点击“取消”：恢复编辑行数据为原始值（或关闭弹窗 ），放弃修改。

### （4）删除解析记录

点击列表行“删除”按钮，调用**删除解析记录接口**：

- 成功：从 `NTable` 数据源中移除对应行，列表自动更新。
- 失败：通过 `NMessage` 组件提示删除失败原因（如权限不足、记录被占用 ）。

## 七、组件样式与细节优化

### （1）响应式适配

通过 `NGrid` 的 `responsive` 属性（或 `NCol` 的 `span` 配置 ），确保在移动端、桌面端下：

- 编辑行的输入框、选择器自动适配宽度，避免挤压变形。
- 表格 `NTable` 支持横向滚动（配置 `horizontal-scroll` ），保证字段完整展示。

### （2）交互反馈

- **按钮点击**：通过 `NButton` 的 `loading` 属性，在接口请求时显示加载态（如保存、删除操作中显示转圈动画 ），增强用户感知。
- **错误提示**：结合 `NMessage` 组件，在接口调用失败时弹出**错误文案**（如 “记录值格式错误，请检查 IP 地址” ），指导用户修正操作。

### （3）状态标签统一

解析记录的状态（待保存、正常、异常 ）通过 `NTag` 组件的 `type` 属性区分颜色：

- 待保存：`type="warning"`（黄色 ）
- 正常：`type="success"`（绿色 ）
- 异常：`type="error"`（红色 ）  
  确保状态可视化，降低理解成本。

通过以上设计，可完整覆盖域名解析场景的**查看、添加、编辑、删除**需求，基于 NaiveUI 组件体系，开发可高效实现页面搭建，保障 DNS 解析管理功能的交互流畅性与视觉一致性。
