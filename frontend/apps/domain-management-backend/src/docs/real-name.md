# 实名模板管理页面需求说明（增加NCard包裹版）

## 一、页面整体布局与模块划分

页面采用简洁的上下布局，从上到下依次为：页面详情区（自定义 `PageHeader`组件）、操作按钮区（NCard包裹）、模板列表区（NCard包裹）、分页区，基于NaiveUI组件搭建，清晰展示实名模板管理相关信息，数据动态加载，与后端交互获取。各功能区块通过卡片组件分隔，增强页面层次感。

## 二、页面详情区（自定义 `PageHeader`组件）

使用用户自定义的 `PageHeader`组件，作为页面头部信息展示区，需传入以下参数：

- **页面标题**：文本为 `实名模板管理`，作为当前页面主标题，字体较大、醒目，突出页面功能。
- **副标题**：文本为 `管理您的实名认证模板，用于域名注册和转移`，对页面功能做补充说明，字体稍小，辅助理解页面用途。

该组件内部包含的元素及交互逻辑，按组件默认行为使用即可。

## 三、操作按钮区（NCard包裹）

使用NaiveUI的 `NCard`组件作为容器，包裹操作按钮，卡片样式保持默认（浅色背景、轻微阴影），去除卡片标题，仅保留内容区域，使按钮区与其他区域形成视觉区分。

### （一）添加新模板按钮

卡片内部放置NaiveUI的 `NButton`组件，按钮文本为 `+ 添加新模板`，类型设为主要按钮（primary），样式为蓝色，点击触发添加新模板的逻辑（如弹出表单弹窗、跳转新页面等，根据业务需求实现），用于创建新的实名模板。按钮靠右对齐，与卡片左侧保持适当间距。

## 四、模板列表区（NCard包裹）

使用NaiveUI的 `NCard`组件作为容器，包裹整个模板列表，卡片样式保持默认（与操作按钮区卡片风格一致），去除卡片标题，仅保留内容区域，使列表区独立成块，提升页面整洁度。

### （一）容器与表头

卡片内部使用NaiveUI的 `NTable`组件承载模板列表，设置表头，包含 `复选框`（可选）、`模板名称`、`类型`、`证件类型`、`证件号码`、`联系电话`、`状态`、`创建时间`、`操作`列，表头文字用 `NText`组件，清晰展示各列含义。表格与卡片边缘保持适当内边距（如16px），避免内容紧贴边框。

### （二）表格数据行

所有数据从后端接口获取，动态渲染：

- **复选框列**：`NCheckbox`组件，用于批量选择模板（如批量删除等操作，若有需求可配置），每行对应一个模板的选择状态。
- **模板名称列**：展示模板对应的名称及关联信息（如示例中 `张三`及邮箱 `zhang****@example.com`），用 `NText`组件，清晰呈现模板命名及辅助信息。
- **类型列**：用 `NTag`组件，根据类型显示不同样式，如个人类型显示浅蓝色标签、企业类型显示浅紫色标签，文本为 `个人`或 `企业`等，区分模板所属主体类型。
- **证件类型列**：展示模板对应的证件类型（如 `身份证`、`营业执照`），`NText`组件显示，明确证件类别。
- **证件号码列**：展示证件号码（做脱敏处理，如 `110********5678`），`NText`组件显示，保护用户信息安全。
- **联系电话列**：展示联系电话（脱敏处理，如 `138****8888`），`NText`组件显示，呈现关键联系信息。
- **状态列**：用 `NTag`组件，显示模板状态（如 `已认证`），设置为绿色标签，突出模板认证状态。
- **创建时间列**：展示模板创建时间（如 `2024-01-15`），`NText`组件显示，记录模板创建节点。
- **操作列**：包含 `查看`和 `删除`操作：
  - `查看`：`NButton`组件，文字为 `查看`，类型为默认或次要按钮，点击触发查看模板详情逻辑（如弹窗展示、跳转详情页）。
  - `删除`：`NButton`组件，文字为 `删除`，类型为错误状态按钮（红色），点击触发删除模板逻辑（需做二次确认等交互，避免误删）。

## 五、分页区

使用NaiveUI的 `NPagination`组件，展示在模板列表区卡片的外部下方，与卡片保持适当间距（如16px），展示分页信息：

- 显示文本：`显示第 X 到 Y 条，共 Z 条记录`，其中 `X`、`Y`、`Z`为接口返回的分页数据（当前页起始条数、当前页结束条数、总条数），动态渲染。
- 分页按钮：包含 `上一页`、`页码`、`下一页`，点击 `上一页`、`下一页`切换页码，加载对应页的模板数据；页码按钮展示当前页及可跳转页码，支持点击切换，实现模板列表的分页浏览。
