# 订单管理页面需求说明（增加PageHeader版）

## 一、页面整体布局与模块划分

页面从上到下依次为：页面详情区（PageHeader）、顶部统计概览区、筛选搜索区、订单列表区 ，整体采用简洁的卡片式、列表式布局，符合 NaiveUI 组件风格，以清晰展示订单管理相关信息。所有数据均为动态加载，随后端返回数据实时更新。

## 二、页面详情区（PageHeader）

使用 NaiveUI 的 `NPageHeader` 组件，作为页面的头部信息展示区，包含以下内容：

- **页面标题**：传入标题文本为"订单管理"，作为当前页面的主标题，字体较大且醒目
- **副标题**：传入副标题文本为"查看和管理所有订单信息"，作为对页面功能的简要说明，字体略小于主标题

该组件内部已包含返回按钮、页面导航等默认元素，无需额外配置，保持组件默认交互行为。

## 三、顶部统计概览区

1. **容器与排版**：用 NaiveUI 的 `NGrid` 或 `NRow` + `NCol` 组件，将统计卡片横向排列，形成一行布局。
2. **统计卡片**：共 4 张卡片，分别对应 “全部订单”“待付款”“已完成”“已取消” ，用 `NCard` 组件实现，所有数字均为动态数据，从后端接口获取：
   - **全部订单卡片**：
     - 左侧图标：`NIcon` 组件，使用购物车相关图标（如 NaiveUI 内置的购物车图标）。
     - 文字内容：“全部订单” 标题 + 动态数字（从接口获取的总订单数），用 `NText` 组件，标题字体稍小、常规，数字字体稍大、突出。
   - **待付款卡片**：
     - 左侧图标：`NIcon` ，用黄色进度条或付款相关图标。
     - 文字内容：“待付款” 标题 + 动态数字（从接口获取的待付款订单数），样式同全部订单卡片，突出待办状态。
   - **已完成卡片**：
     - 左侧图标：`NIcon` ，绿色对勾图标（NaiveUI 内置成功状态图标 ）。
     - 文字内容：“已完成” 标题 + 动态数字（从接口获取的已完成订单数），体现完成状态。
   - **已取消卡片**：
     - 左侧图标：`NIcon` ，红色叉号图标（NaiveUI 内置失败 / 取消状态图标 ）。
     - 文字内容：“已取消” 标题 + 动态数字（从接口获取的已取消订单数），体现取消状态。

## 四、筛选搜索区

1. **布局**：用 `NRow` + `NCol` 组件，将日期选择、搜索框、筛选下拉、搜索按钮横向排列，形成一行操作区。
2. **日期选择**：
   - 两个 `NDatePicker` 组件，类型为日期选择（年 / 月 / 日），分别作为起始日期和结束日期筛选条件，支持点击弹出日历选择，用 `placeholder` 提示 “年 / 月 / 日” 。
3. **搜索框**：`NInput` 组件，`placeholder` 为 “搜索订单号、商品名称...” ，用于输入搜索关键词，输入内容将作为参数传递给后端接口。
4. **筛选下拉**：
   - 两个 `NSelect` 组件，选项数据均从后端接口获取：
     - 第一个 `placeholder` 为 “全部状态” ，选项包含接口返回的所有订单状态（动态生成）。
     - 第二个 `placeholder` 为 “全部类型” ，选项为接口返回的所有订单类型（动态生成）。
5. **搜索按钮**：`NButton` 组件，文字为 “搜索” ，类型可设为主要按钮（primary），点击后将所有筛选条件组合成参数，调用后端接口获取符合条件的订单数据。

## 五、订单列表区

1. **容器与表头**：用 `NTable` 组件，表格数据来源为后端接口返回的订单列表，设置表头，包含 “订单ID”“商品详情”“金额”“状态”“创建时间”“操作” 列，表头文字用 `NText` ，清晰展示各列含义。
2. **表格数据行**：所有数据均为接口返回的订单列表项信息：
   - **订单ID列**：展示订单唯一标识（如接口返回的 "orderId" 字段），用 `NText` ，可根据需求设置是否可点击（如跳转订单详情 ）。
   - **商品详情列**：展示接口返回的商品名称及相关信息（如商品ID、规格等），分两行或一行展示，用 `NText` ，清晰呈现商品内容。
   - **金额列**：展示订单金额（带人民币符号 ¥ ，值为接口返回的 "amount" 字段），`NText` 组件展示，突出金额数值。
   - **状态列**：用 `NTag` 或 `NBadge` 组件，根据接口返回的 "status" 字段显示不同样式：
     - 待处理：黄色标签，文字 “待处理” 。
     - 已完成：绿色标签，文字 “已完成” 。
     - 已取消：红色标签，文字 “已取消” 。
     - 其他状态：根据接口返回的状态值动态显示对应文字和样式。
   - **创建时间列**：展示订单创建的时间戳（接口返回的 "createTime" 字段，格式化后显示），`NText` 组件展示，清晰记录时间。
   - **操作列**：根据接口返回的订单状态（"status" 字段），用 `NButton` 或 `NText` 动态展示可执行的操作：
     - 可取消订单：显示红色 “取消” 按钮（`NButton` ，类型为错误状态 ），点击触发取消订单接口调用。
     - 可支付订单：显示蓝色 “支付” 按钮，点击跳转到支付页面或触发支付接口。
     - 不可操作（如已完成、已取消订单 ）：可不显示按钮或显示灰色不可点击文字（如 “无操作” ，`NText` 设为禁用样式 ）。
     - 其他操作：根据业务需求和订单状态动态显示（如“详情”、“退款”等按钮）。

## 六、底部分页区

显示 “显示第 X 到 Y 条，共 Z 条记录” ，其中 X、Y、Z 均为接口返回的分页信息（当前页起始条数、当前页结束条数、总条数）。用 `NPagination` 组件实现分页功能，动态显示当前页码、总页数，支持点击跳转页码、切换每页条数（选项可配置），页码切换时重新调用列表接口加载对应页的订单数据。
