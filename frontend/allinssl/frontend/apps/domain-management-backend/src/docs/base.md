# 域名详情页需求说明（基于 NaiveUI ）

## 一、页面整体布局与模块划分

页面采用 **“顶部导航 + 标签页 + 卡片化内容”** 结构，基于 NaiveUI 组件搭建，用于展示域名（如 `example.com` ）的详细信息及管理操作。布局从上到下依次为：

- **返回导航区**
- **页面标题区**
- **标签页导航区**（基本信息、实名认证、域名解析、操作日志 ）
- **内容展示区**（按标签页分类渲染对应信息，含域名信息、DNS 解析、实名认证、注册信息、域名安全等模块 ）

## 二、返回导航区

- **组件**：`NButton`（类型为 `text` ，简化样式 ）
- **内容**：文案 “返回域名列表” ，左侧可搭配 `NIcon`（箭头图标，如 `chevron-left` ）
- **交互**：点击跳转回域名列表页，实现页面导航

## 三、页面标题区

- **主标题**：
  - 组件：`NText`（设置较大字号、加粗 ）
  - 内容：域名（如 `example.com` ）
- **副标题**：
  - 组件：`NText`（常规字号、浅色 ）
  - 内容：“域名详细信息和管理”

## 四、标签页导航区

- **组件**：`NTabs` + `NTabPane`
- **标签项**：
  - 基本信息（默认激活 ）、实名认证、域名解析、操作日志
- **交互**：点击标签切换内容区，加载对应模块数据

## 五、内容展示区（以“基本信息”标签页为例）

### 整体容器

用 `NCard` 包裹各信息模块，通过 `NGrid`（或 `NRow` + `NCol` ）做**网格布局**，将内容按区块划分（如 2 列或多列排列 ），提升规整性。

### 模块 1：域名信息

- **容器**：`NCard`（简洁样式，可配置 `bordered` 或 `shadow` ）
- **内容结构**：  
  | 字段 | 值 | 展示组件 |  
  |-------------|-----------------|-------------------|  
  | 域名 | `example.com` | `NText` |  
  | 所有者 | 张三 | `NText` |  
  | 所有者英文 | Zhang San | `NText` |  
  | 域名状态 | 正常（绿色标签） | `NTag`（`type="success"`） |

### 模块 2：DNS 解析

- **容器**：`NCard`
- **内容**：
  - **标题**：`NText`（加粗） + `NIcon`（如 `dns` 图标 ），文案 “DNS 解析”
  - **字段列表**：
    - DNS1：`ns1.example.com`
    - DNS2：`ns2.example.com`
  - **操作按钮**：`NButton`（类型 `primary` ），文案 “修改 DNS 服务器”
    - 交互：点击弹出 DNS 编辑弹窗（或跳转编辑页 ），支持修改 DNS 地址

### 模块 3：实名认证

- **容器**：`NCard`
- **内容结构**：  
  | 字段 | 值 | 展示组件 |  
  |-------------|-----------------|-------------------|  
  | 认证状态 | 已认证（绿色） | `NTag`（`type="success"`） |  
  | 实名类型 | 个人认证 | `NText` |  
  | 域名所有者 | 张三 | `NText` |  
  | 联系人 | 张三 | `NText` |  
  | 手机号码 | 138\***_5678 | `NText` |  
  | 电子邮箱 | zhang_**@example.com | `NText` |

### 模块 4：注册信息

- **容器**：`NCard`
- **内容**：  
  | 字段 | 值 | 展示组件 | 特殊样式 |  
  |-------------|-----------------|-------------------|-----------------------|  
  | 注册时间 | 2023-12-15 | `NText` | - |  
  | 到期时间 | 2024-12-15 | `NText` | 橙色高亮（`type="warning"`） |

### 模块 5：域名安全

- **容器**：`NCard`
- **内容**：
  - **标题**：`NText`（加粗） + `NIcon`（如 `shield` 图标 ），文案 “域名安全”
  - **功能项**：
    - **禁止转移锁**：
      - 状态：已开启（绿色标签 + 勾选框 `NCheckbox` 禁用态 ）
      - 说明：`NText`（“开启后，将禁止域名转出到其他注册商” ）
    - **禁止更新锁**：
      - 状态：未开启（灰色标签 + 勾选框 `NCheckbox` 禁用态 ）
      - 说明：`NText`（“开启后，将禁止更新域名信息...” ）

## 六、其他标签页说明（实名认证、域名解析、操作日志 ）

### 实名认证标签页

- 复用**模块 3 结构**，补充更多实名信息（如证件类型、认证时间等 ），字段与后端接口返回对齐，用 `NText`、`NTag` 等组件展示。

### 域名解析标签页

- 可扩展为**表格 + 操作**形式：
  - 用 `NTable` 展示解析记录（主机记录、记录类型、线路、值、TTL 等 ）
  - 操作列提供 “修改”“删除” 按钮（`NButton` ），点击触发弹窗或接口调用。

### 操作日志标签页

- 用 `NTable` 展示日志列表，字段如：操作时间、操作类型（创建 / 修改 / 删除 ）、操作人、详情 ，支持分页（`NPagination` ）和筛选（`NSelect` + `NInput` ）。

## 七、交互与数据逻辑

- **动态渲染**：所有字段（如域名状态、DNS 地址、实名认证信息 ）从后端接口获取，通过异步请求加载后填充到对应组件。
- **操作交互**：
  - “修改 DNS 服务器” 按钮：点击弹出 `NModal`（模态框 ），内含 `NForm`（输入新 DNS 地址 ），提交后调用接口更新。
  - 标签页切换：`NTabs` 组件默认监听 `change` 事件，可联动接口刷新对应标签页数据（如需实时性 ）。

通过以上结构，可清晰还原域名详情页的布局、组件使用及交互逻辑，开发可基于 NaiveUI 组件快速落地实现。
