# 域名详情 - 操作日志标签页需求说明（基于 NaiveUI ）

## 一、页面整体架构

作为域名详情页的子模块，依托**域名详情页基础框架**，聚焦“操作日志”标签内容展示。采用「顶部导航 + 标签切换 + 日志筛选 + 列表展示 + 分页」布局，基于 NaiveUI 组件构建，实现域名操作历史的分类筛选、详情查看与分页浏览。

## 二、返回导航区

复用域名详情页的返回导航逻辑：  
使用 `NButton`（`text` 类型 ） + `NIcon`（向左箭头 `chevron-left` ），文案为**“返回域名列表”** 。点击触发路由跳转，回到域名列表页，完成页面级导航。

## 三、页面标题区

与域名详情页保持一致：

- **主标题**：显示域名（如 `example.com` ），通过 `NText` 组件设置**大字号、加粗**，突出域名主体。
- **副标题**：“域名详细信息和管理” ，以 `NText` 组件**常规字号、浅色**展示，辅助说明页面功能。

## 四、标签页导航区

基于 `NTabs` 组件实现标签切换：

- 当前激活标签为**“操作日志”** ，通过 `NTabs` 的 `activeKey` 控制选中状态。
- 其他标签（基本信息、实名认证、域名解析 ）保留可切换能力，点击可跳转至对应内容区，支持多维度域名信息的分类查看。

## 五、操作日志标签内容区

### 整体容器

使用 `NCard` 组件包裹所有内容，配置 `bordered` 属性（显示边框 ），为操作日志管理提供规整的视觉容器。通过 `NGrid`（或 `NRow` + `NCol` ）实现布局，按“筛选区 → 日志列表 → 分页区” 纵向排列内容。

### 模块 1：日志筛选区

通过 `NRow` + `NCol` 实现**行内筛选布局**，包含以下组件：

- **操作类型筛选**：`NSelect` 组件，`placeholder` 为**“全部操作”** 。选项从后端接口获取（如 “添加DNS解析记录”“域名续费成功” 等 ），支持动态筛选日志类型。
- **时间范围筛选**：两个 `NDatePicker` 组件（`type="date"` ），分别用于选择**开始时间**和**结束时间**，`placeholder` 为**“年/月/日”** ，支持按时间区间筛选日志。

### 模块 2：操作日志列表

通过 `NList` 组件（或 `NTable` ）展示日志详情，每条日志为一个 `NListItem` ，包含以下内容：

#### （1）日志状态标记

- 使用 `NBadge` 组件，根据操作类型显示不同颜色标记：
  - 添加DNS解析记录：`type="info"`（蓝色 ）
  - 域名续费成功：`type="success"`（绿色 ）
  - 开启禁止转移锁：`type="warning"`（黄色 ）
  - 修改DNS服务器：`type="primary"`（蓝色 ）

#### （2）日志详情内容

每条日志包含：

- **操作标题**：`NText` 组件，加粗展示（如 “添加DNS解析记录”“域名续费成功” ）。
- **操作详情**：`NText` 组件，展示具体操作内容（如 “添加了A记录: @ → 192.168.1.1”“续费1年，新到期时间: 2025-12-15” ）。
- **操作人信息**：`NText` 组件，展示操作人、IP（如 “操作人: 张三 | IP: 192.168.1.100” ）。
- **操作时间**：`NText` 组件，展示日志生成时间（如 “2024-10-15 14:30:25” ），右对齐显示。

### 模块 3：分页区

通过 `NPagination` 组件实现分页功能：

- **数据关联**：与日志列表数据源联动，根据后端接口返回的**总条数**、**当前页条数**动态渲染页码。
- **交互逻辑**：
  - 点击页码（如 `1`、`2`、`3` ）或 “上一页/下一页” 按钮，调用接口加载对应页的操作日志数据，更新 `NList` 内容。
  - 支持配置**每页条数**（如 `page-size` 属性 ），控制单页展示的日志数量。

## 六、数据渲染与交互逻辑

### （1）初始化加载

进入“操作日志”标签页时，调用后端接口获取**第一页操作日志数据**（默认页 ），通过 `NList` 动态渲染日志条目，同时初始化 `NPagination` 的总页数、当前页。

### （2）筛选日志

选择操作类型（`NSelect` ）、时间范围（`NDatePicker` ）后：

- 点击“筛选”按钮（或自动触发 ），调用**带筛选条件的日志查询接口**，参数包含操作类型、开始时间、结束时间。
- 接口返回符合条件的日志列表，更新 `NList` 数据源，同时重置 `NPagination` 为第一页。

### （3）分页浏览

- 点击 `NPagination` 的页码或翻页按钮，调用**分页查询接口**（参数包含当前页、每页条数 ）。
- 接口返回对应页的日志数据，更新 `NList` 内容，`NPagination` 自动同步当前页状态。

## 七、组件样式与细节优化

### （1）响应式适配

通过 `NGrid` 的 `responsive` 属性（或 `NCol` 的 `span` 配置 ），确保在移动端、桌面端下：

- 筛选区的 `NSelect`、`NDatePicker` 自动适配宽度，避免挤压变形。
- 日志列表 `NList` 的条目内容支持换行（如操作详情较长时 ），保证信息完整展示。

### （2）交互反馈

- **筛选操作**：在接口请求时，通过 `NSpin` 组件为 `NCard` 容器添加**加载态**（如转圈动画 ），增强用户感知。
- **分页切换**：点击页码时，`NPagination` 自动显示 `loading` 状态，直到接口返回数据后隐藏。

### （3）日志条目差异化展示

通过 `NBadge` 的颜色区分操作类型，结合 `NList` 的 `meta` 字段自定义日志标题、详情，让不同操作的日志条目在视觉上更易区分（如续费成功用绿色标记，添加解析用蓝色标记 ）。

通过以上设计，可完整覆盖“操作日志”标签页的布局、交互与数据展示逻辑，基于 NaiveUI 组件体系，开发可高效实现页面搭建，保障操作日志的分类筛选、分页浏览功能落地。
